1. Cerință

Sa se creeze o aplicatie Web ce le ofera utilizatorilor diverse tipuri de antrenamente sportive personalizate in functie de greutate, inaltime, varsta, gen, etc.

2. Introducere

Specialiștii au constatat că multe dintre problemele de sănătate apărute pe parcursul vieții pot fi prevenite dacă urmăm un plan alimentar echilibrat și facem mișcare în mod regulat. Beneficiile sportului asupra fizicului ne sunt deja binecunoscute. Însă, trebuie să știm că din ce în ce mai multe studii recente au dovedit că sportul contribuie atât la sănătatea fizică, dar și la cea mentală. Efectuate în mod regulat, activitățile fizice ne îmbunătățesc încrederea în noi și modul în care ne percepem corpul, ne ajută să ne concentram mai ușor și să facem față cu succes stresului.

Ne propunem să facem o aplicație web cu scopul de a ajuta persoanele care sunt in cautarea unui program eficient de exercitii fizice generat conform preferintelor utilizatorului si nevoilor sale.

3. Arhitectura aplicației

Aplicația folosește design pattern-ul MVC, iar diagrama de mai jos descrie pe scurt structura generala:

3.1 Back-end

Backend-ul este construit folosind paradigma MVC.

3.1.1 Baza de date

Baza de date users contine un id unic, un username folosit pentru afisarea clasamentului celor mai activi utilizatori, un email unic, parola, informatiile despre greutate, inaltime, gen, daca doreste sa slabeasca, sa se tonifieze sau sa creasca masa musculara, si care este grupa de muschi cu cel mai mare nivel de interes. Baza de date mai contine si un camp old_weight care retine penultima greutate a utilizatorului care va fi comparata cu cea actuala, dar si un activity_points care se incremeneteaza cand un utilizator termina un workout, si pe baza caruia se realizeaza clasamentul in pagina Statistics.

Cea de-a doua baza de date, images, contine exercitiile folosite pentru generarea workout-ului, cat si informatiile despre numarul de repetari, numarul de pasi si descrierea pasilor, si de asemenea un mod corect de respiratie pentru utilizator in timpul exercitiului curent.

3.1.2 Php

Cu ajutorul limbajului de programare destinat backend-ului se realizeaza conexiunea la baza de date, cat si manipularea acesteia pentru a extrage/introduce datele utilizatorului curent. Tot cu ajutorul acestui limbaj, se transmit datele prin sesiune pentru a ne asigura ca modificarile, cat si clasamentul se realizeaza din perspectiva utilizatorului logat.

3.2 Front-end

3.2.1 Start

Pagina Start ofera posibilitatea ca un nou utilizator sa isi creeze cont sau utilizatorii cu un cont existent sa se logheze, dar si un link catre informatii referitoare la proprietarii site-ului.

3.2.2 Create account

În această pagină, utilizatorul isi poate face un cont introducand datele personale.

3.2.3 Login

In aceasta pagina utilizatorul se poate loga folosind email-ul si parola.

3.2.4 Reset password

In aceasta pagina utilizatorul isi poate reseta parola in cazul in care nu si-o mai aminteste.

3.2.5 Profile

Dupa ce utilizatorul s-a logat, sau si-a creat contul, va fi redirectionat catre pagina profilului sau, unde isi poate vedea atat informatiile publice, cat si cele private si respectiv le poate modifica.

3.2.6 Edit profile

In acesta pagina utilizatorul isi poate modifica efectiv informatiile.

3.2.7 Generator

In acesta pagina utilizatorul alege ce grupe de muschi ar vrea sa lucreze in ziua respectiva, locatia unde se desfasoara workout-ul, dar si durata acestuia.

3.2.8 Workout

Dupa ce si-a creat workout-ul in pagina "Start workout", utilizatorul este redirectionat catre pagina aceasta, unde sunt prezentate exercitiile generate impreuna cu instructiuni detaliate si exemple foto.

3.2.9 Informations

Aceasta pagina contine informatii despre sport si cateva beneficii ale acestuia asupra corpului nostru.

3.2.10 Statistics

In pagina Statistics se poate gasi un top al celor mai activi utilizatori, dar si niste statistici personale ale utilizatorului curent, care ulterior pot fi descarcate in formatele JSON si PDF.

4. Mod de functionare

  1. Utilizatorul are optiunea de a-si crea un cont nou, sau de a se loga.
  2. Dupa conectare, se memoreaza intr-o variabila email pastrata in sesiunea curenta emailul utilizatorului curent, iar acesta este redirectionat catre pagina Profil unde isi poate vizualiza datele, atat in Profil, cat si in pagina Statistici.
  3. Daca doreste sa se antreneze, utilizatorul navigheaza catre pagina Generator, unde isi introduce preferintele.
  4. In pagina Workout cu ajutorul unui apel Ajax se afiseaza primul exercitiu, iar la apasarea butonului "next exercise" se realizeaza de fiecare data un nou apel ajax.
  5. Dupa finalizarea workout-ului, apasarea butonului de "finish workout" se incrementeaza activity_points-ul userului curent, iar acesta este redirectionat catre pagina Generator de unde poate incepe un nou workout.
  6. Cand userul doreste sa se delogheze, butonul de "Logout" din meniu va distruge datele sesiunii si acesta este redirectionat catre Login.

5. Tehnologii utilizate

Limbaje de programare precum : JavaScript (pentru interactivitate, pentru procesarea informatiilor prezentate), CSS (pentru prezentarea informatiilor), HTML5 (pentru structura semantica a informatiilor), PHP.

Tehnica de programare Ajax pentru crearea unei aplicatii interactive folosind limbajul JavaScript este folosita pentru a afisa mesaje de eroare ce informeaza utilizatorul ca nu este logat, ca informatiile introduse nu sunt corecte, ca mail-ul pentru resetarea parolei a fost trimis cu succes, etc. Tehnica Ajax mai este folosita si pentru a afisa exercitiile generate pentru workout-ul curent in maniera interactiva, fara reincarcarea paginii, cu un simplu click de mouse. Acest lucru este posibil prin folosirea multipla a apelurilor ajax, realizat in cascada.
